# Hazard Detection Fine-Tune — Dataset Specification

> **Status: SCAFFOLD ONLY — no dataset collected, model NOT trained.**
> This document specifies what training data would be needed to extend KÖREM's
> YOLOv8n model with fine-grained outdoor hazard classes. Collection and
> annotation remain as future work.

---

## Target hazard classes

These are the 8 hazard types prioritised for blind/wheelchair users in the
article. They augment the existing 80 COCO classes without replacing them.

| ID  | EN label        | RU label              | Priority |
|-----|-----------------|-----------------------|----------|
| 80  | stairs_up       | ступени вверх         | HIGH     |
| 81  | stairs_down     | ступени вниз          | HIGH     |
| 82  | curb            | бордюр                | HIGH     |
| 83  | open_manhole    | открытый люк          | HIGH     |
| 84  | pole            | столб / опора         | MEDIUM   |
| 85  | construction    | строительный барьер   | MEDIUM   |
| 86  | wet_floor       | мокрый пол / лужа     | MEDIUM   |
| 87  | sign_board      | знак / вывеска        | LOW      |

## Annotation format

YOLO v8 `.txt` per image, one object per line:
```
<class_id> <cx_norm> <cy_norm> <w_norm> <h_norm>
```
- All values in \[0, 1\] relative to image dimensions.
- Class IDs start at 80 (appended after COCO-80).

## Directory layout

```
training/data/
  images/
    train/   ← ≥ 150 images per class recommended
    val/     ← 20% split
  labels/
    train/
    val/
  dataset.yaml   ← generated by train_yolo_hazard.py
```

## Capture guidelines

1. All photos from ~1.2 m height (cane / phone height for standing user).
2. Vary lighting: daylight, dusk, overcast, indoor artificial.
3. Vary distance: 0.5 m – 5 m from hazard.
4. Include partial occlusion (e.g. pole partially behind pedestrian).
5. Avoid motion blur above ~50 ms exposure.
6. Annotate **tight** bounding boxes — do not include background in box.

## Annotation tool

Recommended: [Roboflow](https://roboflow.com) (web UI + export YOLO v8).
Alternative: [CVAT](https://cvat.ai).

Export format: YOLOv8 (includes `dataset.yaml` automatically).

## Data augmentation (applied during training)

Controlled by `train_yolo_hazard.py`. Default augmentations:
- Horizontal flip (p=0.5)
- Mosaic (p=0.8)
- HSV shift (h=0.015, s=0.7, v=0.4)
- Random crop / scale (0.5–1.5)
- No vertical flip (stairs_up ≠ stairs_down after vertical flip)
